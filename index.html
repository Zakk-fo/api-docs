Estimate Landed Cost

Get duty, VAT, excise and the all-in landed cost for any item — from a plain description (Zakk classifies it with AI) or a known HS code. Price a single product or a whole shipment in one call. This guide walks you through it end to end.

STEP 1

Get an access token

Every call needs a bearer token. Exchange your Zakk username and password for one using the OAuth 2.0 password grant at POST /api/auth/token. The token is short-lived — check expires_in (seconds) and refresh it with the returned refresh_token before it expires.

authenticate.sh 
curl -s -X POST https://zakk.app/api/auth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=password" \
  -d "username=$ZAKK_USERNAME" \
  -d "password=$ZAKK_PASSWORD"
200 OK · response 
{
  "access_token": "eyJhbGciOiJIUzI1NiIs…",   // use this as the Bearer token
  "refresh_token": "def50200a1b2…",
  "token_type": "Bearer",
  "expires_in": 3600,
  "is_email_verified": true
}
Base URLs. Production is https://zakk.app. Staging (if enabled for your account) is https://dev.zakk.app. Send access_token on every request as Authorization: Bearer <token>.

STEP 2

Compose your request

The body is always a { "lines": [ … ] } array — one element per item. For each line, give Zakk either an item description (it classifies for you) or a known HS code, plus a destination. Add a customs value and measurements to get a full landed-cost figure. Fill the form below and watch the request build itself.

request body · live
Empty fields are dropped. Send only what you have. With no value you get a rates-only answer; add value + the right measurement (quantity / weightKg) for a full landed cost. To price a shipment, push more objects into lines — up to 100.

STEP 4

Read the response

You get a batch envelope: lineCount, successCount, errorCount, then a lines array that mirrors your input order. Each line carries its own status and either a result or an error.

200 OK · response (abridged) 
{
  "lineCount": 1, "successCount": 1, "errorCount": 0,
  "lines": [{
    "index": 0, "reference": "PO-1001", "status": "ok",
    "result": {
      "authority": "FO", "isClassified": false,
      "estimates": [{
        "hsCode": "61091000", "confidence": 99, "qualityState": "Provided",
        "isPreferential": false,
        "landedCost": {
          "customsValue": 1500,
          "dutyAmount": 150, "dutyExpression": "10%",
          "vatAmount": 442.5,
          "exciseAmount": 0,
          "freightCost": 120, "includesFreight": true,
          "totalLandedCost": 2212.5
        },
        "landedCostDecision": { "outcome": "calculated", /* components, … */ },
        "warnings": []
      }]
    }
  }]
}
landedCost.totalLandedCostdecimal
The headline number: customs value + duty + excise + VAT (+ freight), all in the destination authority's settlement currency. This is what you quote.
dutyAmountdecimal | null
0 means a genuine 0% / exempt duty. null means the rate couldn't be verified — duty is then excluded from the total, so it reads as a "from" lower bound. Check landedCostDecision.outcome for "partial".
isPreferentialboolean
true when an originCountry you supplied qualified for a preferential rate. When that beats the MFN rate, savings shows the difference.
qualityStatestring
Verified, Adapted, NotFound… Tells you how trustworthy the matched tariff line is. Pair it with warnings.
landedCostDecision.outcomestring
calculated (complete), partial (a rate was unknown or a measurement was missing), blocked (need more input), or unsupported (no tariff coverage). Drives the badge in the Zakk UI.
warnings[]{ code, message }
Machine-readable nudges — e.g. MissingAlcoholVolumePercent, CodeAdapted. Branch on code; show message to humans.
Per-line errors are isolated. A line that can't be priced comes back with "status":"error", the errorStatusCode a single-line request would have returned, a stable errorCode (e.g. no_valid_codes_for_authority), and a human-readable error. The rest of the batch still succeeds. Aggregate authorities (EU, BASE) aren't supported and return a per-line error.

STEP 5

Field reference

Every field a line accepts. Provide either itemDescription or hsCode, never both.

FieldTypeNotes
destinationCountrystringrequiredISO 3166-1 alpha-2. Supported: FO, NO, GB, CA, US. EU/BASE rejected per line.
itemDescriptionstringeither/orFree text. Triggers AI classification — may return several ranked estimates.
hsCodestringeither/orDirect tariff lookup. Dots/spaces/dashes are stripped. Digit length depends on the authority (FO/NO = 8, GB/CA/US = 10).
valuedecimaloptionalCustoms value. Required to compute a landed cost; omit for a rates-only answer.
currencystringoptionalISO-4217 of value & freightCost. Converted to the authority's settlement currency where applicable.
quantitydecimaloptionalUnits. Needed for per-unit duties and per-volume alcohol excise.
weightKgdecimaloptionalNet weight in kg. Needed for weight-based duties.
originCountrystringoptionalISO alpha-2. Unlocks preferential rates + savings when a trade agreement covers it.
freightCostdecimaloptionalAdded to the VAT base when present; surfaced as includesFreight: true.
alcoholVolumePercentdecimaloptional0–100. Required for per-vol% alcohol excise (e.g. Norwegian spirits/wine). Omit → excise stays 0 + a warning.
supplementaryQuantitydecimaloptionalVolume in litres/HL for GB-style specific excise charged per unit of volume.
referencestringoptionalYour own id (SKU, PO line). Echoed back verbatim so you can stitch results to your rows.
Discover supported destinations at runtime. Call GET /api/hscodes/destinations and check each entry's hasLandedCostSupport: true flag rather than hard-coding the list. The legacy route POST /api/hscodes/resolve-tariff is a name alias that takes the same lines body.
